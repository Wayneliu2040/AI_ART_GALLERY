# 详细任务拆解版开发计划

## 规划原则

本计划完全遵循 proposal 中的技术约束，并以“先完成可运行 MVP，再完善部署和展示材料”为核心思路。

项目交付目标：

- 本地可以运行
- 核心功能端到端可联调
- 能部署到 Azure
- 适合课程项目展示与答辩

## 阶段 1：项目初始化与协作规范

### 主要任务

- 建立仓库目录结构
  - `frontend/`
  - `backend/`
  - `docs/`
- 确认技术版本
- 确定分支策略
  - `main`
  - `dev`
  - 功能分支
- 完善 README 和协作文档
- 统一命名规范
  - API 路径命名
  - 数据库表命名
  - Azure 资源命名
- 准备环境配置文件

### 阶段交付物

- 目录结构清晰
- 前后端项目都能本地启动
- 团队协作方式固定

## 阶段 2：MVP 范围冻结

### MVP 必做功能

- 注册
- 登录
- 上传图片
- 浏览全部图片
- 搜索与筛选
- 查看图片详情
- 评论
- 点赞

### 非 MVP 功能

- 个人主页
- 删除图片
- 编辑图片信息
- 热门排序
- 复杂分页

### 阶段交付物

- 一份固定的 MVP 功能列表
- 每个功能有明确的验收标准

## 阶段 3：数据库与对象存储设计

### Azure SQL Database 任务

- 绘制 ER 图
- 设计数据表结构
- 编写 Migration 或建表 SQL
- 设置主键和外键
- 为常用查询字段添加索引

### 推荐数据表

#### `Users`

- `Id`
- `Username`
- `Email`
- `PasswordHash`
- `CreatedAt`

#### `Images`

- `Id`
- `UserId`
- `Title`
- `Description`
- `Prompt`
- `Tags`
- `BlobUrl`
- `CreatedAt`

#### `Comments`

- `Id`
- `ImageId`
- `UserId`
- `Content`
- `CreatedAt`

#### `Likes`

- `Id`
- `ImageId`
- `UserId`
- `CreatedAt`

### Azure Blob Storage 任务

- 创建 Storage Account
- 创建 Blob 容器，例如 `ai-images`
- 约定文件命名方式
- 明确图片访问策略
- 仅在 SQL 中存储 Blob URL

### 阶段交付物

- 稳定的数据模型
- 可用的 Azure SQL 数据库
- 可用的 Blob 容器

## 阶段 4：后端开发

## 4.1 后端基础设施

### 主要任务

- 创建 ASP.NET Core Web API 项目
- 配置项目目录结构：
  - Controllers
  - Services
  - Repositories
  - Models
  - DTOs
  - Data
- 配置 EF Core 与 Azure SQL 连接
- 配置 CORS
- 配置 Swagger
- 增加统一异常处理
- 增加文件上传大小限制

### 阶段交付物

- API 本地可运行
- Swagger 可访问
- 数据库连接正常

## 4.2 认证模块

### 主要任务

- 实现 `POST /api/auth/register`
- 实现 `POST /api/auth/login`
- 对密码进行哈希处理
- 返回 JWT 或课程项目简化 token
- 给需要登录的接口加认证保护
- 处理邮箱重复、密码错误等异常情况

### 阶段交付物

- 注册功能可用
- 登录功能可用
- 受保护接口可以验证用户身份

## 4.3 图片上传模块

### 主要任务

- 实现 `POST /api/images`
- 接收 multipart/form-data
- 校验文件类型和文件大小
- 上传图片到 Azure Blob Storage
- 将图片元数据和 Blob URL 写入 Azure SQL
- 返回创建成功的图片信息

### 阶段交付物

- 图片上传链路完整可用
- Blob 中可看到上传文件
- SQL 中可看到对应元数据

## 4.4 图片浏览模块

### 主要任务

- 实现 `GET /api/images`
- 按时间倒序返回图片列表
- 实现 `GET /api/images/{id}`
- 区分列表返回字段和详情返回字段

### 阶段交付物

- 图片列表接口可用
- 图片详情接口可用

## 4.5 搜索与筛选模块

### 主要任务

- 实现 `GET /api/images/search?q=...&tag=...`
- 支持标题、描述、prompt、tag 搜索
- 定义空查询时的返回规则
- 保持匹配逻辑一致

### 阶段交付物

- 搜索接口可直接演示

## 4.6 评论与点赞模块

### 主要任务

- 实现 `POST /api/images/{id}/comments`
- 实现 `GET /api/images/{id}/comments`
- 实现 `POST /api/images/{id}/like`
- 处理重复点赞问题
- 必要时返回当前点赞数

### 阶段交付物

- 评论可以新增和展示
- 点赞可以提交并反映到页面

## 阶段 5：前端开发

## 5.1 前端基础结构

### 主要任务

- 初始化 React 项目
- 配置路由
- 创建页面结构：
  - Login
  - Register
  - Gallery
  - ImageDetail
  - Upload
  - MyImages
- 提取公共组件：
  - Navbar
  - ImageCard
  - SearchBar
  - CommentList
  - UploadForm
- 封装 API 请求方法
- 配置环境变量

### 阶段交付物

- React 工程结构清晰
- 路由切换正常

## 5.2 登录与注册页面

### 主要任务

- 登录页接入 `/api/auth/login`
- 注册页接入 `/api/auth/register`
- 保存 token
- 登录成功后跳转
- 增加受保护路由逻辑
- 增加表单校验和错误提示

### 阶段交付物

- 前端认证流程完整

## 5.3 画廊页面

### 主要任务

- 从 `/api/images` 加载真实数据
- 展示图片卡片
- 显示标题、标签、prompt 摘要和点赞数
- 添加搜索栏
- 添加标签筛选
- 将搜索与后端接口联动

### 阶段交付物

- 画廊页展示真实数据
- 搜索与筛选可用

## 5.4 上传页面

### 主要任务

- 增加文件选择器
- 增加标题、描述、prompt、tag 输入项
- 提交到 `/api/images`
- 显示加载、成功、失败反馈
- 上传成功后跳转或刷新页面

### 阶段交付物

- 前端上传功能可用

## 5.5 图片详情页

### 主要任务

- 从 `/api/images/{id}` 加载详情
- 展示图片和全部元数据
- 展示评论列表
- 支持发表评论
- 支持点赞交互

### 阶段交付物

- 图片详情页满足演示需求

## 5.6 体验完善

### 主要任务

- 增加加载状态
- 增加空状态
- 增加错误状态
- 优化表单校验反馈
- 处理图片加载失败占位

### 阶段交付物

- 页面更完整、更稳定

## 阶段 6：前后端联调

### 主要任务

- 统一请求和响应结构
- 统一错误返回格式
- 明确 token 的传递方式
- 明确上传请求结构
- 统一前后端字段命名
- 记录并修复联调问题

### 阶段交付物

- 前端不再依赖 mock 数据
- 核心流程端到端跑通

### 阶段交付物

### 阶段交付物

- 后端公网地址可访问

## 7.3 Azure SQL 与 Blob 接入

### 主要任务

- 配置数据库防火墙规则
- 设置生产环境连接字符串
- 执行 Migration 或建表脚本

### 阶段交付物

- 生产环境 API 能正常访问 SQL 和 Blob

### 主要任务

- 验证云端部署环境

### 阶段交付物

- 测试 checklist
- 稳定可演示的版本

### 主要任务

## 推荐开发顺序

为了尽快得到一个能跑的版本，建议按以下顺序推进：

1. 项目初始化
2. Azure SQL 和 Blob 搭建
3. 注册和登录 API
4. 上传 API
5. 图片列表 API
6. 前端登录与画廊联调
7. 详情页、评论和点赞
8. 搜索与筛选
9. Azure 部署
10. 测试与最终文档整理
